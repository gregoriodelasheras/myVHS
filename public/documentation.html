Documentation

Objective:

Build a web application provides users with access to information about different movies, directors and genres released during the 1980s. Users are able to register, update their personal information and create a list of their favorite movies.

Key Features:

  • Return a list of all movies to the user.
  • Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user.
  • Return data about a genre (description) by name/title (e.g., “Thriller”).
  • Return data about a director (bio, birth year, death year) by name.
  • Allow new users to register.
  • Allow users to update their user info (username, password, email, date of birth).
  • Allow users to add a movie to their list of favorites.
  • Allow users to remove a movie from their list of favorites.
  • Allow existing users to deregister.
  • Allow users to see which actors star in which movies.
  • Allow users to view information about different actors.
  • Allow users to view more information about different movies, such as the release date and the movie rating.
  • Allow users to create a “To Watch” list in addition to their “Favorite Movies” list.

Built With

  • MongoDB
  • Mongoose
  • Express
  • React
  • Node.js
  • Visual Studio Code
  • Prettier
  • ESLint

Endpoints Design

# Business Logic URL HTTP Method Request Response
1 Return a list of all movies to the user. /movies GET None A JSON object holding data about all the movies.
2 Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user. /movies/[title] GET None A JSON object holding data about a single movie queried, containing a title, director, releaseDate, runtime, genre and mainCast.
3 Allow users to see which actors star in which movies. /movies/[title]/maincast GET None A JSON object holding about a single movie queried, containing the list of the actors stars acting in it.
4 Allow users to view more information about different movies (Writers, Producers, Music Direction, Art Direction and the Movie Rating). /movies/[title]/extradata GET None A JSON object holding extra data about a single movie queried, containing a writers, producers, musicDirection, artDirection and movieRating,
5 Return data about a genre (description) by name/title (e.g., “Thriller”). /genre/[name] GET None A JSON object holding data about the film genre queried.
6 Return data about a director (bio, birth year, death year, works) by name. /director/[name] GET None A JSON object holding data about the film director queried, containing a name, birthYear, deathYear, bio and works.
7 Allow users to view information about different actors. /actors/[name] GET None A JSON object holding data about the actor queried, containing a name, birthYear, deathYear, bio and works.
8 Allow new users to register. /users POST A JSON object holding data about the user to add, containing an ID, username, password, name, email and date of birth. A JSON object holding data about the user that was added, containing id, username, password, name, email and birthDate.
9 Allow users to update their user info (username, password, email, date of birth) through its ID. /users/[id] PUT None A text message indicating the data that was updated.
10 Allow existing users to deregister. /users/[id] DELETE None A text message indicating the name of the user who was deleted from the database.
11 Allow users to add a movie to their list of favorites. /users/[id]/favmovies POST None A text message indicating the name of the movie that was added to the "Favorite Movies" list.
12 Allow users to remove a movie from their list of favorites. /users/[id]/favmovies DELETE None A text message indicating the name of the movie that was removed from the "Favorite Movies" list.
13 Allow users to create a "To Watch" list in addition to their "Favorite Movies" list. /users/[id]/towatch POST None A text message indicating the name of the movie that was added to the "To Watch" list.
